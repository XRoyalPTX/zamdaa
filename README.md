# Joloo (Backend) 

Асинхронный бэкенд для сервиса поиска попутчиков (межрайонное такси). Проект разработан как полноценный командный кейс для портфолио с акцентом на чистую архитектуру, асинхронность и готовность к интеграции с фронтендом/мобильным приложением.

## 🛠 Стек технологий

*   **Фреймворк:** FastAPI (Python 3.11+) — асинхронный, высокопроизводительный.
*   **База данных:** PostgreSQL 16 (в Docker-контейнере).
*   **ORM:** SQLAlchemy 2.0 (Async Engine) + Alembic для миграций.
*   **Контейнеризация:** Docker + Docker Compose.
*   **Валидация данных:** Pydantic v2.

## Быстрый запуск для фронтенд / мобильных разработчиков

Бэкенд полностью изолирован и упакован в Docker. Вам не нужно устанавливать Python, базы данных или настраивать окружение локально.

### 1. Клонирование репозитория и окружение
Заберите код и создайте файл настроек (если требуется, скопируйте из примера):
```bash
git clone [https://github.com/твоё_имя/joloo.git](https://github.com/твоё_имя/joloo.git)
cd joloo
```

### 2. Запуск Docker-контейнеров
Убедитесь, что у вас запущен Docker Desktop, и поднимите сервисы бэкенда и базы данных в фоновом режиме:
```bash
docker compose up -d
```
*   API Бэкенда доступно по адресу: ```http://localhost:8000```
*   Интерактивный Swagger (Документация): ```http://localhost:8000/docs```

### 3. Наполнение базы данных населёнными пунктами Бурятии
Чтобы в приложении сразу появились реальные районы, города и улусы для создания маршрутов, запустите скрипт импорта данных прямо внутри контейнера:
```bash
docker compose exec app python seed.py
```
Скрипт автоматически отфильтрует и зальет в PostgreSQL 734 населенных пункта Республики Бурятия с правильной привязкой к районам (аймакам) и типам поселений.
